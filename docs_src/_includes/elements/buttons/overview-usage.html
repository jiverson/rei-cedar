Buttons are meant for transactional use. They can be considered the "utilitarian" buttons, seen within product focused areas of the site, like Search Results, Product Details, or login pages. These buttons help the user act on an informational, research or purchase path of an REI product.

Button examples: "Submit", "Checkout", "Add to cart", "See more", "Continue shopping".

Use primary buttons for actions to complete a task or to move forward in a process such as "Add to cart." There is only one primary action per major page section.

Button sizes and shapes

Change the button size based on where the button is used. The default size is medium. Small is used for supplemental user actions such as product comparison or filter. Large is used for "Add to cart" on product pages or for Call to Action.

Use the full breakpoint width in XS grid. Width: mobile version extends to full breakpoint size (Minus gutter). Text: sentence case(horizontally centered within button).

In specific cases (recommended for hub or campaign pages only) This default behavior can be overridden, resulting in a narrower style identical to larger breakpoints. Alignment should align with parent style. Text: sentence case(horizontally centered within button).
