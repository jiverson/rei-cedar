Images

Responsive Images

Images can be made responsive-friendly via the addition of the .img-responsive class. This applies max-width: 100%;, height: auto; and display: block; to the image so that it scales nicely to the parent element.

To center images which use the .img-responsive class, use .center-block instead of .text-center. See the helper classes section for more details about .center-block usage.

Overlays

Add classes to an <img> element to easily style images in any project.

Cross-browser compatibility

Keep in mind that Internet Explorer 8 lacks support for rounded corners.

Aspect Ratio

Utilize the .utility-aspect-ratio mixin or .ratio-[ square, 1-2, 2-3, 3-4, 9-16, 2-1, 3-2, 4-3, 16-9 ] class to set an element with no implicit height ( such as the case when its contents are positioned absoluetely ).

Applying .content-[ top, center, bottom ] (or using modifier --content-[ top, center, bottom ] will align contents within the container.

Components

Image / Media Frame

Images can be scaled (cropped or constrained) to a container element by applying to the parent an .img-frame class. This applies position: relative;, overflow: hidden; and padding-bottom: [ Variable ]; to maintain the proportions of a flexible-width element. Applying a secondary class of .ratio-[ square, 1-2, 2-3, 3-4, 9-16, 2-1, 3-2, 4-3, 16-9 ] will set it to a desired aspect ratio. When the mixin is invoked directly, it allows for a set pixel height.

Images contained within a parent container with class .media-frame( Preferred ) or .img-frame will be set to apply position: absolute;, width: auto;, height: auto;.

With an additional class of .center, .top, .right, .bottom, or .left, the image will be achored to its respective side. min-height: 100%;, min-width: 100%;, object-fit: cover;, and either height: 100%; (.left, .right, or .center) or width: 100%; (.top, .bottom, or .portrait.center). When the mixin is invoked directly, percent and pixel values can be applied for more fine-tuned image anchoring.

Note: Center-anchored frames are not aspect-agnostic:

Please see above: If you are centering a portrait image, it will require an additional class .portrait to ensure it is correctly anchored.

With no positioning classes set (or with optional class .nocrop set), it will apply a max-width: 100%;, max-height: 100%;, object-fit: contain; and use .utility-position to center within the container. Wells or padding can be applied to the container to add space between the edges and the image.




Responsive embed

Allow browsers to determine video or slideshow dimensions based on the width of their containing block by creating an intrinsic ratio that will properly scale on any device.

Rules are directly applied to <iframe>, <embed>, <video>, and <object> elements; optionally use an explicit descendant class .embed-responsive-item when you want to match the styling for other attributes.

Pro-Tip! You don't need to include frameborder="0" in your <iframe>s as we override that for you.
